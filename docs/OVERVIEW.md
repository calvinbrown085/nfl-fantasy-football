# NFL Fantasy Schedule Parsers - Developer Overview

This document provides a technical overview for developers working with the NFL Fantasy Schedule Parsers project.

## Project Description
This project comprises Python scripts designed for web scraping and parsing NFL.com Fantasy Football league data. It allows users to retrieve either a detailed season schedule for their individual team or a comprehensive view of league-wide matchups, including past results, current week's starting lineups, and fantasy point projections. The core functionality relies on fetching dynamic HTML content from NFL.com and extracting specific data points using robust HTML parsing techniques.

## Architecture
The project follows a modular, client-side web scraping architecture:
-   **Core Library (`nfl_common.py`):** This module acts as the backend, centralizing reusable functions for HTTP requests (`requests`), HTML parsing (`BeautifulSoup4` with `lxml`), and specific data extraction routines (e.g., team IDs, player rosters, owner names, matchup details).
-   **Application Scripts (`my_team_schedule.py`, `league_matchups.py`):** These scripts serve as the frontend applications. They import and orchestrate functions from `nfl_common.py` to perform specific data retrieval and display tasks:
    -   `my_team_schedule.py`: Focuses on an individual team's schedule, showing past results, the current week's game, and upcoming matchups.
    -   `league_matchups.py`: Provides a league-wide view, displaying previous week's results and current week's matchups, complete with owner names, detailed starting lineups, and projected fantasy points.
-   **Authentication:** Access to personalized or league-specific data on NFL.com Fantasy requires authentication, which is handled by manually configured session cookies passed in HTTP headers with each request.
-   **Configuration:** Essential parameters such as the target fantasy league ID are hardcoded into the URLs used by the scripts.

## Key Files

*   `nfl_common.py`: This is the core utility module. It contains all shared functions for network requests (e.g., `fetch_schedule_html`), HTML parsing (e.g., `parse_current_week_matchups`, `parse_team_roster`), and data extraction (e.g., `extract_team_ids_from_matchups`, `extract_team_owner`, `extract_team_name`).
*   `my_team_schedule.py`: The main script for fetching and displaying an individual fantasy team's season schedule. It uses functions from `nfl_common.py` to parse schedule data and presents past results, current week information, and upcoming games.
*   `league_matchups.py`: The main script for fetching and displaying league-wide information. It leverages `nfl_common.py` to parse previous week's results and current week's matchups, including owner names and detailed starting lineups with fantasy point projections.
*   `pyproject.toml`: Defines the project's metadata, Python version requirement (`>=3.10`), and specifies the primary dependencies (`requests`, `beautifulsoup4`, `lxml`) required for the project. Used by `uv` for dependency management.
*   `uv.lock`: A lock file generated by the `uv` dependency manager, ensuring consistent and reproducible installations of project dependencies across different environments.
*   `resources/`: A directory intended to store local HTML files (e.g., `schedule.html`, `team.html`). These files can be used for offline testing of the parsing logic without making live network requests to NFL.com.

## How to Run

### Prerequisites
-   Python 3.10 or higher.
-   `uv` (recommended) or `pip` for dependency management.

### Installation
1.  **Install `uv` (if not already installed):**
    ```bash
    curl -LsSf https://astral.sh/uv/install.sh | sh
    ```
2.  **Install project dependencies:**
    ```bash
    uv sync
    ```
    Alternatively, using `pip`:
    ```bash
    pip install requests beautifulsoup4 lxml
    ```

### Execution
-   **Personal Team Schedule:**
    ```bash
    uv run my_team_schedule.py
    ```
-   **League-Wide Matchups:**
    ```bash
    uv run league_matchups.py
    ```

## Configuration

The scripts require specific configuration to function correctly and access personalized league data:

*   **Authentication Cookies:** Both `my_team_schedule.py` and `league_matchups.py` contain a `cookies` variable (e.g., `cookies = ''`). This string **must be updated** with your active NFL.com session cookies to authenticate requests and access private league data. Without valid cookies, the scripts will likely fail to fetch personalized or league-specific information.
*   **League ID:** The `base_url` or `url` variable in both `my_team_schedule.py` and `league_matchups.py` is hardcoded with a specific league ID (e.g., `12698811`). To use the scripts with a different fantasy league, this ID **must be manually updated** in the URL string within both scripts.

## How to Test

The parsing logic within the `nfl_common.py` module can be tested independently of live web requests:

1.  **Local HTML Files:** The `resources/` directory is designed to hold sample HTML outputs (e.g., `schedule.html`, `team.html`) captured from NFL.com Fantasy pages. These files provide static data for testing.
2.  **Direct Function Calls:** You can import and call parsing functions directly from `nfl_common.py` into a separate test script or an interactive Python session (e.g., using `python -i` or an IDE's debug console).
3.  **Example Testing Scenario:**
    ```python
    # Example for testing nfl_common.py functions
    from nfl_common import parse_current_week_matchups, parse_team_roster
    
    # Load a local HTML file containing a league schedule
    try:
        with open('resources/schedule.html', 'r', encoding='utf-8') as f:
            html_content = f.read()
        matchups = parse_current_week_matchups(html_content)
        print("Parsed Matchups:", matchups)
    except FileNotFoundError:
        print("resources/schedule.html not found. Please create it with sample HTML.")
    
    # Load a local HTML file containing a team's page
    try:
        with open('resources/team.html', 'r', encoding='utf-8') as f:
            team_html_content = f.read()
        roster = parse_team_roster(team_html_content)
        print("Parsed Roster:", roster)
    except FileNotFoundError:
        print("resources/team.html not found. Please create it with sample HTML.")
    ```
    This approach allows developers to verify parsing accuracy against known HTML structures, debug parsing issues, and develop new features without making repeated network calls or being dependent on the live NFL.com website, which might have rate limits or frequently changing content.
